

# Plan: Fix In-App Purchase Flow for Apple App Store Approval

## Root Cause Analysis

The "Something went wrong" error comes from line 83-85 of `Subscribe.tsx` — the `catch` block around `StoreKit.showPaywall()`. After thorough analysis, there are **three critical issues** in the JavaScript-to-native bridge:

1. **No product ID is passed to native code.** The user selects a plan in the web UI (e.g. "Weekly Updates" with productId `com.physiquecrafters.app.monthly`), but `showPaywall()` takes zero arguments. The native side has no idea which product to purchase.

2. **User cancellation is treated as an error.** When StoreKit 2 purchase is cancelled by the user, the native side likely rejects the promise. The JS catch block shows "Something went wrong" — even for a normal cancellation. Apple testers cancel and retry; this error toast likely contributed to rejection.

3. **No native Swift plugin code exists in this repo.** The `ios/` directory doesn't exist here (it's generated by `npx cap add ios`). I'll provide the complete Swift plugin code you need to add to your Xcode project alongside the JS fixes.

**Key native Swift logic (reference for Xcode):**
```swift
@objc func purchase(_ call: CAPPluginCall) {
    guard let productId = call.getString("productId") else {
        call.reject("Missing productId")
        return
    }
    Task {
        do {
            let products = try await Product.products(for: [productId])
            guard let product = products.first else {
                call.reject("PRODUCT_NOT_FOUND")
                return
            }
            let result = try await product.purchase()
            switch result {
            case .success(let verification):
                let transaction = try checkVerified(verification)
                await transaction.finish()
                call.resolve(["success": true, "productId": productId])
            case .userCancelled:
                call.reject("USER_CANCELLED")
            case .pending:
                call.reject("PURCHASE_PENDING")
            @unknown default:
                call.reject("PURCHASE_FAILED")
            }
        } catch {
            call.reject("PURCHASE_FAILED", error.localizedDescription)
        }
    }
}
```
