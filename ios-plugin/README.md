# StoreKit Plugin — Xcode Integration Guide

## Prerequisites
- iOS 15+ target
- Products created in App Store Connect with IDs:
  - `com.physiquecrafters.app.monthly`
  - `com.physiquecrafters.app.biweekly`
  - `com.physiquecrafters.app.training`
- Xcode 15+

## Step 1: Add the Swift file

1. Open your Xcode project (generated by `npx cap add ios`)
2. Navigate to `App/App/` in the project navigator
3. Create a folder called `Plugins` (right-click → New Group)
4. Drag `StoreKitPlugin.swift` into the `Plugins` folder
5. Make sure "Copy items if needed" is checked and the target `App` is selected

## Step 2: Register the plugin in AppDelegate.swift

Open `App/App/AppDelegate.swift` and add the plugin registration:

```swift
import UIKit
import Capacitor

@UIApplicationMain
class AppDelegate: UIResponder, UIApplicationDelegate {

    var window: UIWindow?

    func application(_ application: UIApplication,
                     didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?) -> Bool {
        return true
    }

    // Register the StoreKit plugin with Capacitor
    func application(_ app: UIApplication, open url: URL,
                     options: [UIApplication.OpenURLOptionsKey: Any] = [:]) -> Bool {
        return ApplicationDelegateProxy.shared.application(app, open: url, options: options)
    }
}
```

Then in `App/App/Bridge/` or wherever your bridge is initialized, ensure the plugin is auto-discovered. Capacitor 5+ auto-discovers `CAPBridgedPlugin` conforming classes — no manual registration needed.

If auto-discovery doesn't work, add this to your `AppDelegate`:

```swift
func applicationDidBecomeActive(_ application: UIApplication) {
    if let bridge = (window?.rootViewController as? CAPBridgeViewController)?.bridge {
        bridge.registerPluginInstance(StoreKitPlugin())
    }
}
```

## Step 3: Enable StoreKit in Xcode

1. Select your project target → **Signing & Capabilities**
2. Click **+ Capability** → add **In-App Purchase**

## Step 4: Configure products in App Store Connect

1. Go to [App Store Connect](https://appstoreconnect.apple.com)
2. Navigate to your app → **Subscriptions**
3. Create a Subscription Group (e.g. "Coaching Plans")
4. Add three auto-renewable subscriptions:
   - `com.physiquecrafters.app.monthly` — $399.99/month
   - `com.physiquecrafters.app.biweekly` — $299.99/month
   - `com.physiquecrafters.app.training` — $174.99/2 months
5. Set status to **Ready to Submit** for each

## Step 5: Test in Sandbox

1. In App Store Connect → **Users and Access** → **Sandbox Testers**, create a test account
2. On your test device, sign out of your real Apple ID in Settings → App Store
3. Run the app via Xcode, sign in with the sandbox account when prompted
4. The purchase sheet will show "[Environment: Sandbox]"

## Step 6: Sync & Build

```bash
npx cap sync ios
# Then open in Xcode:
npx cap open ios
# Build and run on device/simulator
```

## Troubleshooting

| Issue | Fix |
|-------|-----|
| "Product not found" | Ensure product IDs match exactly in App Store Connect, and products are in "Ready to Submit" status |
| Plugin not found at runtime | Verify `StoreKitPlugin.swift` is included in the App target's "Compile Sources" build phase |
| Purchase sheet doesn't appear | Must test on a real device or Xcode simulator with sandbox account |
| "Something went wrong" in app | Check Xcode console for the specific error code from the native plugin |
