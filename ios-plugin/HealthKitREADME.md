# HealthKit Plugin — Xcode Integration Guide

## Prerequisites
- iOS 13+ target (HealthKit APIs used are available from iOS 13)
- Xcode 15+
- Physical device recommended (HealthKit has limited simulator support)

## Step 1: Add the Swift file

1. Open your Xcode project (generated by `npx cap add ios`)
2. Navigate to `App/App/` in the project navigator
3. Create a folder called `Plugins` (right-click → New Group) if it doesn't exist
4. Drag `HealthKitPlugin.swift` into the `Plugins` folder
5. Make sure "Copy items if needed" is checked and the target `App` is selected

## Step 2: Enable HealthKit Capability

1. Select your project target → **Signing & Capabilities**
2. Click **+ Capability** → add **HealthKit**
3. Under the HealthKit capability, ensure **Clinical Health Records** is **unchecked** (we only need basic health data)

## Step 3: Add Usage Description to Info.plist

Open `App/App/Info.plist` and add:

```xml
<key>NSHealthShareUsageDescription</key>
<string>Physique Crafters reads your step count, active calories, walking distance, and body weight from Apple Health to track your fitness progress and sync with your coach.</string>
```

This string is displayed in the HealthKit authorization dialog. Apple will reject your app if this key is missing.

## Step 4: Register the Plugin (if auto-discovery fails)

Capacitor 5+ auto-discovers `CAPBridgedPlugin` conforming classes. If the plugin isn't found at runtime, add manual registration in `AppDelegate.swift`:

```swift
func applicationDidBecomeActive(_ application: UIApplication) {
    if let bridge = (window?.rootViewController as? CAPBridgeViewController)?.bridge {
        bridge.registerPluginInstance(HealthKitPlugin())
    }
}
```

## Step 5: Sync & Build

```bash
npx cap sync ios
npx cap open ios
# Build and run on device
```

## What Data We Read (Read-Only)

| HealthKit Type | Usage |
|----------------|-------|
| `HKQuantityType.stepCount` | Daily step tracking, challenges |
| `HKQuantityType.activeEnergyBurned` | Active calorie tracking |
| `HKQuantityType.distanceWalkingRunning` | Distance in km |
| `HKQuantityType.bodyMass` | Weight tracking (most recent) |

We do **not** write any data to HealthKit. Read-only access minimizes App Store review friction.

## Troubleshooting

| Issue | Fix |
|-------|-----|
| "HealthKit is not available" | HealthKit isn't supported on iPad (some models) or simulator without health data |
| Authorization dialog doesn't appear | Check that the HealthKit capability is added and `NSHealthShareUsageDescription` is in Info.plist |
| Plugin not found at runtime | Verify `HealthKitPlugin.swift` is in the App target's "Compile Sources" build phase |
| Steps return 0 | User may have denied permission — HealthKit returns empty data (not an error) when denied |
| "Clinical Health Records" warning | Make sure only basic HealthKit is enabled, not Clinical Records |

## Privacy & Compliance

- Apple requires that HealthKit data is used **only for health/fitness purposes** — our coaching use case qualifies
- HealthKit data must **never** be shared with third parties or used for advertising
- The Privacy Policy at `/privacy-policy` includes Section 9b covering health data collection
- Users can revoke access anytime via iOS Settings → Privacy & Security → Health → Physique Crafters
